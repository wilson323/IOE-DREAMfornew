# 变更: 实现SmartAdmin公共模块体系

## 为什么
基于 docs/COMMON_MODULES/ 文档分析，SmartAdmin项目需要实现一套完整的公共模块体系来支撑各业务模块的开发。当前项目中虽然有详细的设计文档，但缺少实际代码实现，导致业务模块开发时需要重复造轮子，影响开发效率和代码质量。

## 变更内容
- **统一设备管理模块** (smart-device): 支持摄像头、门禁、消费终端、考勤机等设备统一管理
- **门禁业务模块** (smart-access): 依赖设备模块，专注门禁业务逻辑和权限管理
- **权限管理增强** (smart-permission): 5级安全级别权限控制
- **地理位置模块** (smart-location): GPS定位、地理围栏、位置验证
- **实时数据模块** (smart-realtime): WebSocket实时通信、数据推送、缓存管理
- **区域管理模块** (smart-area): 区域层级管理、区域权限、区域统计
- **人员信息模块** (smart-person): 人员档案、人脸特征、权限关联

## 技术架构
- **基础架构**: Spring Boot 3 + 四层架构 + Sa-Token + Redis
- **设备管理**: 策略模式 + 工厂模式 + 协议适配器
- **实时通信**: WebSocket + 消息队列 + 事件发布订阅
- **权限控制**: RBAC + 数据权限 + 5级安全级别
- **缓存策略**: L1(Caffeine) + L2(Redis) 多级缓存

## 影响范围
- **影响规格**: 无 (新增独立模块)
- **影响代码**:
  - 后端: 新增 7 个公共模块包结构
  - 前端: 新增对应的 API、Store、组件
  - 数据库: 新增公共模块相关表结构
- **影响部署**: 需要配置 Redis、WebSocket、消息队列等基础设施
