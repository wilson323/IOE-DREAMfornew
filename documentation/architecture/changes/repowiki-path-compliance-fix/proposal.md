# IOE-DREAM项目repowiki路径规范合规化修复提案

## Why

基于深度分析，IOE-DREAM项目在repowiki路径规范方面存在重大不合规问题：

### 🔴 重大不合规问题
- **异常嵌套路径**: 错误的嵌套路径结构影响编译
- **Jakarta迁移不彻底**: 8个文件仍使用javax包（一级违规）
- **DAO命名不统一**: Repository和Dao混用

### 🟡 中等不合规问题
- **前端API目录重复**: consume和consumption并存
- **前端模块分散**: 缺乏统一的组织标准
- **模块粒度不一致**: 前后端结构不对应

### 📊 影响评估
- **维护成本增加**: 不规范结构增加维护难度
- **开发效率降低**: 团队缺乏统一标准
- **技术债务累积**: 不符合规范导致技术债务
- **团队协作困难**: 标准不统一影响协作

## 变更概述

本变更旨在系统性地解决IOE-DREAM项目在repowiki路径规范方面的重大不合规问题，确保项目严格遵循`.qoder/repowiki`规范体系，提升代码质量和维护性。

## 问题分析

基于深度分析发现以下关键问题：

### 🔴 重大不合规问题（立即修复）

1. **异常嵌套路径结构** - 严重违反项目结构规范
2. **Jakarta迁移不彻底** - 8个文件仍使用javax包（一级违规）
3. **DAO层命名不统一** - 混用Repository和Dao命名

### 🟡 中等不合规问题（建议修复）

4. **前端API目录重复** - consume和consumption并存
5. **前端模块分散** - 业务模块组织不一致
6. **模块粒度不统一** - 缺乏统一的组织方式

4. **提升开发效率** - 通过规范统一减少认知负担
