## 技术债识别方法

使用技术债识别脚本定期扫描：

```powershell
.\scripts\identify-technical-debt.ps1 -Detailed
```

脚本会生成：
- Markdown清单: `documentation/project/TECHNICAL_DEBT.md`
- JSON报告: `documentation/project/technical-debt_*.json`

## 技术债管理原则

1. **新代码零容忍**: 新代码必须符合所有规范，不允许引入新的技术债
2. **旧代码逐步修复**: 历史技术债按优先级逐步偿还
3. **定期审查**: 每周审查技术债清单，更新状态
4. **量化跟踪**: 定期统计技术债数量和趋势
