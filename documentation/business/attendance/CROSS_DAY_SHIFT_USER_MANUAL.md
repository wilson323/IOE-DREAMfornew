## 1. 功能概述

### 1.1 什么是跨天班次？

跨天班次是指**上班时间在某一天，下班时间在第二天**的工作班次。例如：

- **夜班A**：22:00 上班 → 次日 06:00 下班
- **早班B**：05:00 上班 → 14:00 下班
- **保安班**：20:00 上班 → 次日 08:00 下班

### 1.2 核心功能

IOE-DREAM考勤系统提供完整的跨天班次支持：

✅ **自动检测** - 系统自动识别跨天班次
✅ **灵活配置** - 支持三种跨天归属规则
✅ **智能统计** - 自动计算工作时长和加班时长
✅ **准确考勤** - 确保跨天打卡记录正确归属

### 1.3 适用场景

| 行业 | 典型场景 | 示例班次 |
|------|---------|---------|
| **制造业** | 夜班生产 | 22:00-06:00 |
| **医疗** | 医护值班 | 20:00-08:00 |
| **安保** | 保安执勤 | 18:00-06:00 |
| **交通** | 公交司机 | 05:30-14:00 |
| **服务业** | 24小时客服 | 轮班制 |

## 3. 核心概念

### 3.1 跨天标识（isOvernight）

**定义**：标记班次是否跨越自然日（00:00）

```
示例1：正常班次 09:00-18:00
├─ 上班时间：09:00（当天）
├─ 下班时间：18:00（当天）
└─ isOvernight = false（不跨天）

示例2：跨天班次 22:00-06:00
├─ 上班时间：22:00（当天）
├─ 下班时间：06:00（次日）
└─ isOvernight = true（跨天）✅
```

### 3.2 跨天规则（crossDayRule）

系统提供三种跨天归属规则：

#### 规则1：START_DATE（以开始日期为准）

**原则**：所有打卡记录都归属到**班次开始日期**

```
夜班：22:00 上班 → 次日 06:00 下班

├─ 21:55 上班打卡 → 归属到 1月1日 ✅
├─ 22:00 上班打卡 → 归属到 1月1日 ✅
├─ 06:00 下班打卡 → 归属到 1月1日 ✅
└─ 06:05 下班打卡 → 归属到 1月1日 ✅

结果：1月1日的考勤记录包含4条打卡记录
```

**适用场景**：
- ✅ 夜班考勤统计
- ✅ 班次完整性要求高
- ✅ 按班次统计工时

#### 规则2：END_DATE（以结束日期为准）

**原则**：所有打卡记录都归属到**班次结束日期**

```
夜班：22:00 上班 → 次日 06:00 下班

├─ 21:55 上班打卡 → 归属到 1月2日 ✅
├─ 22:00 上班打卡 → 归属到 1月2日 ✅
├─ 06:00 下班打卡 → 归属到 1月2日 ✅
└─ 06:05 下班打卡 → 归属到 1月2日 ✅

结果：1月2日的考勤记录包含4条打卡记录
```

**适用场景**：
- ✅ 按自然日统计出勤
- ✅ 夜班作为次日工作
- ✅ 与财务结算周期对齐

#### 规则3：SPLIT（分别归属）

**原则**：打卡记录按**实际打卡日期**归属

```
夜班：22:00 上班 → 次日 06:00 下班

├─ 21:55 上班打卡 → 归属到 1月1日 ✅
├─ 22:00 上班打卡 → 归属到 1月1日 ✅
├─ 06:00 下班打卡 → 归属到 1月2日 ✅
└─ 06:05 下班打卡 → 归属到 1月2日 ✅

结果：1月1日有2条打卡记录，1月2日有2条打卡记录
```

**适用场景**：
- ✅ 按自然日统计工时
- ✅ 需要拆分跨天班次
- ✅ 灵活排班管理

## 4. 操作指南

### 4.1 创建跨天班次

#### 步骤1：进入班次管理页面

```
考勤管理 → 基础信息 → 班次管理
```

#### 步骤2：点击"新增班次"

![新增班次按钮](images/add-shift-button.png)

#### 步骤3：填写班次基本信息

| 字段 | 说明 | 示例 |
|------|------|------|
| **班次名称** | 班次的名称 | 夜班A |
| **班次类型** | 选择类型 | 固定班次 |
| **上班时间** | 选择时间 | 22:00 |
| **下班时间** | 选择时间 | 06:00 |

#### 步骤4：系统自动检测跨天

当您选择的下班时间早于上班时间时，系统会：

1. **自动标记为跨天班次** ✅
2. **显示警告提示** 📢
3. **要求选择跨天规则** ⚠️

```
┌─────────────────────────────────────────┐
│  ⚠️ 检测到跨天班次                        │
│  该班次的下班时间早于上班时间，            │
│  系统已自动标记为跨天班次。                │
│  请选择跨天归属规则。                      │
└─────────────────────────────────────────┘
```

#### 步骤5：选择跨天规则

系统提供三种规则供选择：

```
跨天规则: [下拉选择框]

├─ 以开始日期为准（推荐）⭐
│  └─ 所有打卡记录归属到班次开始日期
│
├─ 以结束日期为准
│  └─ 所有打卡记录归属到班次结束日期
│
└─ 分别归属
   └─ 上班打卡归属当天，下班打卡归属次日
```

**推荐选择**：**以开始日期为准**（START_DATE）

**原因**：
- ✅ 符合夜班考勤习惯
- ✅ 班次数据完整
- ✅ 工时计算准确

#### 步骤6：完善其他信息

| 字段 | 说明 | 推荐值 |
|------|------|--------|
| **工作时长** | 自动计算 | 8.0 小时 |
| **休息时长** | 休息时间 | 0 分钟 |
| **弹性时间** | 允许弹性 | 0 分钟 |
| **排序号** | 显示顺序 | 2 |

#### 步骤7：保存班次

点击"确定"按钮，系统会验证：

✅ 上班时间已填写
✅ 下班时间已填写
✅ 跨天班次已选择跨天规则
✅ 工作时长合理

验证通过后，班次创建成功！

### 4.2 编辑跨天班次

#### 步骤1：找到要编辑的班次

在班次列表中找到目标班次，点击"编辑"按钮。

#### 步骤2：修改班次信息

**可修改的字段**：
- 班次名称
- 班次类型
- 上班/下班时间
- **跨天规则**（可更改）
- 工作时长
- 休息时长

**⚠️ 注意**：修改跨天规则会影响历史考勤记录的归属！

#### 步骤3：保存修改

点击"确定"保存修改。

**系统提示**：
```
修改跨天规则会影响历史考勤记录的统计，
是否确认修改？

[取消]  [确认修改]
```

### 4.3 删除跨天班次

**⚠️ 重要提示**：删除班次前，请确保：

1. ✅ 没有员工正在使用该班次
2. ✅ 没有关联的考勤记录
3. ✅ 没有未完成的排班计划

**步骤**：
1. 点击"删除"按钮
2. 系统会检查班次使用情况
3. 确认后删除

**系统检查项**：
```
正在检查班次使用情况...

✅ 检查员工班次分配
✅ 检查考勤记录关联
✅ 检查排班计划

检查完成，可以安全删除
```

## 7. 常见问题

### Q1: 跨天班次的加班时间如何计算？

**A**: 系统自动计算加班时间，规则如下：

```java
加班时长 = 实际工作时长 - 标准工作时长

示例:
标准工作: 22:00 - 06:00 (8小时)
实际打卡: 21:50 - 06:15
实际工作: 8小时25分钟
加班时长: 25分钟 ✅
```

### Q2: 员工夜班多次打卡怎么办？

**A**: 系统会自动处理：

```
员工打卡记录:
├─ 21:50 第一次上班打卡 → 采用
├─ 22:05 第二次上班打卡 → 忽略（重复打卡）
├─ 06:00 第一次下班打卡 → 采用
└─ 06:10 第二次下班打卡 → 忽略（重复打卡）

结果: 只采用第一次和第三次打卡 ✅
```

### Q3: 修改跨天规则会影响历史数据吗？

**A**: **会的！** 请谨慎操作。

```
修改规则前:
├─ 2025-01-01 夜班: 考勤归属到 1月1日
└─ 2025-01-02 夜班: 考勤归属到 1月2日

修改规则后（START_DATE → END_DATE）:
├─ 2025-01-01 夜班: 考勤归属到 1月2日 ⚠️
└─ 2025-01-02 夜班: 考勤归属到 1月3日 ⚠️

影响: 历史考勤统计会发生变化！
```

**建议**：
- ✅ 尽量在班次创建时确定规则
- ✅ 修改前导出历史考勤数据备份
- ✅ 修改后重新生成考勤报表

### Q4: 跨天班次的休息时间如何设置？

**A**: 在班次配置中设置：

```yaml
班次: 生产夜班 (22:00 - 06:00)
休息时长: 60分钟

系统计算:
├─ 总时长: 8小时 (22:00 - 06:00)
├─ 休息时间: 60分钟
└─ 工作时长: 7小时 (480分钟)

考勤统计:
├─ 实际工作7小时，无加班 ✅
└─ 如工作7.5小时，加班30分钟 ✅
```

### Q5: SPLIT规则会导致缺卡吗？

**A**: **是的！** 需要特殊处理。

```
SPLIT规则下的打卡记录:
├─ 2025-01-01: 只有上班打卡 → 缺下班打卡 ❌
└─ 2025-01-02: 只有下班打卡 → 缺上班打卡 ❌

解决方案:
方案1: 使用START_DATE或END_DATE规则（推荐）
方案2: 开发专门算法处理SPLIT班次
方案3: 人工调整考勤记录
```

**强烈推荐**：使用START_DATE或END_DATE规则，避免数据拆分问题。

### Q6: 如何验证跨天班次配置正确？

**A**: 三步验证法：

```
步骤1: 查看班次列表
✅ 跨天标识列显示"是"
✅ 跨天规则列显示选择的规则

步骤2: 测试打卡
✅ 模拟夜班打卡（22:00上班，06:00下班）
✅ 查看考勤记录归属日期是否正确

步骤3: 查看统计报表
✅ 日报: 夜班出勤是否统计到班次开始日期
✅ 月报: 月度出勤天数是否正确
```

### Q7: 员工请假如何处理跨天班次？

```yaml
夜班: 22:00 - 06:00 (START_DATE规则)

考勤处理:
```

## 8. 最佳实践

### 8.1 配置建议

#### ✅ 推荐做法

1. **统一使用START_DATE规则**
   ```
   适用99%的夜班场景
   - 数据完整性高
   - 统计逻辑简单
   - 员工易于理解
   ```

2. **班次命名清晰**
   ```yaml
   ✅ 好的命名:
   - 生产夜班A (22:00-06:00)
   - 护士夜班 (20:00-08:00)
   - 保安夜班 (18:00-06:00)

   ❌ 不好的命名:
   - 夜班1
   - N1
   - 22-06班
   ```

3. **设置合理的弹性时间**
   ```yaml
   推荐配置:
   上班弹性: +15分钟（避免迟到）
   下班弹性: +30分钟（考虑交接）
   ```

4. **配置休息时长**
   ```yaml
   生产夜班: 0-30分钟（不停工）
   护士夜班: 60-120分钟（轮换休息）
   保安夜班: 120分钟（包含用餐）
   ```

#### ❌ 避免做法

1. **避免使用SPLIT规则**
   ```
   除非有特殊需求，否则不推荐使用
   - 数据拆分导致统计复杂
   - 容易出现缺卡问题
   - 需要额外算法处理
   ```

2. **避免频繁修改跨天规则**
   ```
   修改规则会影响历史数据
   - 导致考勤统计不一致
   - 需要重新生成报表
   - 可能引发员工投诉
   ```

3. **避免忽略休息时长**
   ```
   不设置休息时长会导致:
   - 工作时长计算不准确
   - 加班统计错误
   - 薪资计算争议
   ```

### 8.2 管理建议

#### 定期检查

**每月检查项**：
- [ ] 跨天班次配置是否合理
- [ ] 考勤记录归属是否正确
- [ ] 工时计算是否准确
- [ ] 加班统计是否符合预期

**检查方法**：
```
1. 导出考勤明细报表
2. 筛选跨天班次记录
3. 验证考勤日期归属
4. 对比实际打卡时间
```

#### 员工培训

**培训要点**：
1. **夜班打卡时间**
   ```
   提前5-10分钟打卡
   避免过早打卡（>30分钟）
   ```

2. **跨天班次理解**
   ```
   夜班考勤归属到上班当天
   下班打卡虽然在次日，但仍算当天班次
   ```

3. **异常处理**
   ```
   打卡失败立即报告
   遗漏打卡及时补卡
   设备故障联系管理员
   ```

```
每月归档备份:

```

### 8.3 故障排查

#### 常见问题及解决

**问题1：考勤记录归属错误**

```
症状: 夜班下班打卡归属到次日
原因: 跨天规则配置错误
解决:
1. 检查班次跨天规则设置
2. 修改为正确的规则（推荐START_DATE）
3. 重新生成考勤报表
```

**问题2：工作时长计算错误**

```
症状: 工作时长与实际不符
原因: 未设置休息时长
解决:
1. 在班次配置中设置休息时长
2. 系统会自动计算: 工作时长 = 总时长 - 休息时长
3. 重新统计工时
```

**问题3：加班统计不准确**

```
症状: 加班时间过多或过少
原因: 弹性时间设置不当
解决:
1. 检查弹性时间配置
2. 调整为合理的值（上班+15，下班+30）
3. 重新计算加班时长
```

### A. 术语表

| 术语 | 说明 |
|------|------|
| **跨天班次** | 上班时间和下班时间不在同一天的工作班次 |
| **跨天标识** | 标记班次是否跨天的字段（isOvernight） |
| **跨天规则** | 决定跨天打卡记录归属日期的规则 |
| **START_DATE** | 以班次开始日期为准的跨天规则 |
| **END_DATE** | 以班次结束日期为准的跨天规则 |
| **SPLIT** | 分别归属的跨天规则（按实际打卡日期） |
| **工作时长** | 员工实际工作的时长（扣除休息） |
| **弹性时间** | 允许员工提前或延迟打卡的时间范围 |
