# SmartAdmin v3 门禁系统模块实施计划

> **📋 计划类型**: 详细实施计划
> **📋 模块名称**: smart-access (门禁系统模块)
> **📋 创建时间**: 2025-01-13
> **📋 计划周期**: 7周 (2025年1月第3周 - 2025年3月第2周)
> **📋 项目规模**: 7个数据表，6大功能模块，前后端+移动端全栈实现

---

## 📊 项目概览

### 项目背景
SmartAdmin v3 项目已成功实现了 `smart-device` 设备管理模块，为门禁系统提供了设备管理基础。现需基于现有基础，实现完整的门禁系统公共模块，提供统一的门禁设备管理、精细化权限控制、实时监控和远程控制等功能。

### 项目目标
- ✅ **功能完整**: 实现完整的门禁管理系统功能
- ✅ **架构优秀**: 遵循SmartAdmin四层架构规范
- ✅ **性能卓越**: 满足高并发和实时性要求
- ✅ **安全可靠**: 通过全面的安全测试验证
- ✅ **易于扩展**: 支持多种设备类型和协议扩展

### 核心价值
- **提升安全性**: 统一权限控制，提升园区安全水平
- **提高效率**: 自动化管理，减少人工操作成本
- **增强体验**: 实时监控和便捷控制，提升用户体验
- **数据驱动**: 完整的数据分析，支持业务决策

## 🎯 实施范围详细说明

### 1. 数据库设计实现

#### 1.1 核心业务表设计
```sql
-- 1. 门禁设备表 (smart_access_device)
CREATE TABLE smart_access_device (
    device_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_code VARCHAR(100) NOT NULL UNIQUE,
    device_name VARCHAR(200) NOT NULL,
    device_type VARCHAR(50) NOT NULL, -- door/gate/barrier/turnstile
    device_brand VARCHAR(100),
    device_model VARCHAR(100),
    area_id BIGINT,
    area_name VARCHAR(100),
    location_desc VARCHAR(500),
    ip_address VARCHAR(50),
    port INT,
    mac_address VARCHAR(50),
    protocol_type VARCHAR(50), -- tcp/udp/http/websocket
    status TINYINT DEFAULT 1, -- 0-离线，1-在线，2-故障
    is_enabled TINYINT DEFAULT 1,
    config_json JSON,
    last_online_time DATETIME,
    created_by BIGINT NOT NULL,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_by BIGINT,
    updated_time DATETIME ON UPDATE CURRENT_TIMESTAMP
);

-- 2. 门禁权限表 (smart_access_permission)
CREATE TABLE smart_access_permission (
    permission_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    person_id BIGINT NOT NULL,
    person_name VARCHAR(100) NOT NULL,
    person_type VARCHAR(50) NOT NULL, -- employee/visitor/contractor
    device_id BIGINT NOT NULL,
    device_name VARCHAR(200) NOT NULL,
    area_id BIGINT,
    area_name VARCHAR(100),
    permission_type VARCHAR(50) NOT NULL, -- permanent/temporary/scheduled
    start_time DATETIME NOT NULL,
    end_time DATETIME NOT NULL,
    monday_access TINYINT DEFAULT 1,
    tuesday_access TINYINT DEFAULT 1,
    wednesday_access TINYINT DEFAULT 1,
    thursday_access TINYINT DEFAULT 1,
    friday_access TINYINT DEFAULT 1,
    saturday_access TINYINT DEFAULT 0,
    sunday_access TINYINT DEFAULT 0,
    time_config JSON,
    status TINYINT DEFAULT 1, -- 0-禁用，1-启用，2-待审批，3-已过期
    business_code VARCHAR(100),
    approve_user_id BIGINT,
    approve_time DATETIME,
    approve_remark TEXT,
    created_by BIGINT NOT NULL,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_by BIGINT,
    updated_time DATETIME ON UPDATE CURRENT_TIMESTAMP
);

-- 3. 通行记录表 (smart_access_record)
CREATE TABLE smart_access_record (
    record_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_id BIGINT NOT NULL,
    device_name VARCHAR(200) NOT NULL,
    device_code VARCHAR(100) NOT NULL,
    person_id BIGINT NOT NULL,
    person_name VARCHAR(100) NOT NULL,
    person_type VARCHAR(50) NOT NULL,
    card_number VARCHAR(100),
    face_image_url VARCHAR(500),
    fingerprint_template VARCHAR(500),
    access_result TINYINT NOT NULL, -- 0-失败，1-成功
    fail_reason VARCHAR(200),
    direction VARCHAR(20), -- in/out
    access_time DATETIME NOT NULL,
    temperature DECIMAL(4,1),
    mask_detected TINYINT,
    image_urls JSON,
    event_type VARCHAR(50), -- normal/abnormal/forced
    is_abnormal TINYINT DEFAULT 0,
    processed TINYINT DEFAULT 0,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 4. 设备指令表 (smart_access_command)
CREATE TABLE smart_access_command (
    command_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_id BIGINT NOT NULL,
    command_type VARCHAR(50) NOT NULL, -- open/close/restart/sync
    command_content JSON,
    command_status TINYINT DEFAULT 0, -- 0-待发送，1-已发送，2-执行成功，3-执行失败
    send_time DATETIME,
    execute_time DATETIME,
    response_content JSON,
    error_message TEXT,
    retry_count INT DEFAULT 0,
    max_retry INT DEFAULT 3,
    created_by BIGINT NOT NULL,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 5. 设备状态表 (smart_access_device_status)
CREATE TABLE smart_access_device_status (
    status_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_id BIGINT NOT NULL UNIQUE,
    door_status VARCHAR(20), -- open/closed/unknown
    lock_status VARCHAR(20), -- locked/unlocked/unknown
    power_status VARCHAR(20), -- normal/low/offline
    network_status VARCHAR(20), -- online/offline
    last_heartbeat DATETIME,
    cpu_usage DECIMAL(5,2),
    memory_usage DECIMAL(5,2),
    storage_usage DECIMAL(5,2),
    temperature DECIMAL(4,1),
    error_count INT DEFAULT 0,
    last_error_time DATETIME,
    last_error_message TEXT,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 6. 区域权限组表 (smart_access_area_group)
CREATE TABLE smart_access_area_group (
    group_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    group_name VARCHAR(200) NOT NULL,
    group_code VARCHAR(100) NOT NULL UNIQUE,
    description TEXT,
    area_ids TEXT,
    device_ids TEXT,
    default_start_time TIME,
    default_end_time TIME,
    default_weekdays VARCHAR(20) DEFAULT '1,2,3,4,5',
    status TINYINT DEFAULT 1,
    created_by BIGINT NOT NULL,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_by BIGINT,
    updated_time DATETIME ON UPDATE CURRENT_TIMESTAMP
);

-- 7. 设备告警表 (smart_access_alarm)
CREATE TABLE smart_access_alarm (
    alarm_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    device_id BIGINT NOT NULL,
    alarm_type VARCHAR(50) NOT NULL, -- offline/forced/door_open/low_power/error
    alarm_level VARCHAR(20) NOT NULL, -- low/medium/high/critical
    alarm_title VARCHAR(200) NOT NULL,
    alarm_content TEXT,
    alarm_time DATETIME NOT NULL,
    record_id BIGINT,
    person_id BIGINT,
    person_name VARCHAR(100),
    is_handled TINYINT DEFAULT 0,
    handled_by BIGINT,
    handled_time DATETIME,
    handle_remark TEXT,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 2. 技术架构设计

#### 2.1 后端四层架构
```
Controller层 (接口控制)
├── AccessDeviceController (设备管理接口)
├── AccessPermissionController (权限管理接口)
├── AccessRecordController (通行记录接口)
├── AccessControlController (设备控制接口)
└── AccessMonitorController (监控告警接口)

Service层 (业务逻辑)
├── AccessDeviceService (设备管理业务)
├── AccessPermissionService (权限管理业务)
├── AccessRecordService (通行记录业务)
├── AccessControlService (设备控制业务)
├── AccessMonitorService (监控告警业务)
└── AccessStatisticsService (数据统计业务)

Manager层 (复杂业务)
├── DeviceConnectionManager (设备连接管理)
├── AccessPermissionValidator (权限验证器)
├── AccessEventListener (事件监听器)
├── DeviceProtocolFactory (协议工厂)
└── AlarmProcessor (告警处理器)

DAO层 (数据访问)
├── AccessDeviceMapper (设备数据访问)
├── AccessPermissionMapper (权限数据访问)
├── AccessRecordMapper (通行记录数据访问)
├── AccessCommandMapper (指令数据访问)
└── AccessAlarmMapper (告警数据访问)
```

#### 2.2 前端页面结构
```
src/views/access/
├── device/
│   ├── DeviceList.vue (设备列表页面)
│   ├── DeviceDetail.vue (设备详情页面)
│   └── DeviceControl.vue (设备控制页面)
├── permission/
│   ├── PermissionList.vue (权限列表页面)
│   ├── PermissionApply.vue (权限申请页面)
│   └── PermissionApproval.vue (权限审批页面)
├── record/
│   ├── RecordList.vue (通行记录页面)
│   └── RecordStatistics.vue (统计分析页面)
├── monitor/
│   ├── RealTimeMonitor.vue (实时监控页面)
│   └── AlarmManagement.vue (告警管理页面)
└── config/
    ├── AreaGroupConfig.vue (区域权限组配置)
    └── SystemConfig.vue (系统配置页面)
```
