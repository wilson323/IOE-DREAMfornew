# 🚨 AI助手快速决策指南 - 禁止重蹈覆辙

## ⚡ 紧急风险评估原则

### 🚫 绝对禁止的操作（零容忍）
1. **Lombok注解修改** - 高风险链式反应
2. **大规模代码重写** - 不可预测的连锁错误
3. **依赖结构调整** - 破坏性编译影响
4. **编码格式批量修改** - UTF-8污染风险

### ✅ 安全操作清单
1. **单文件修改** - 确保影响范围可控
2. **逐个编译验证** - 每修改立即检查结果
3. **备份策略** - git restore随时可用
4. **渐进式修复** - 一次只修复一类问题

## 🎯 错误数量对应策略

### 🔴 1-10个错误：精确修复模式
- **策略**：逐一分析，精确修复
- **验证**：每修复1个立即编译验证
- **技能调用**：`Skill("compilation-error-prevention-specialist")`

### 🟡 11-50个错误：分类修复模式
- **策略**：按类型批量修复
- **验证**：每修复10个编译验证一次
- **技能调用**：`Skill("systematic-compilation-error-specialist")`

### 🔴 50+个错误：系统重构模式
- **策略**：暂停修复，分析根源问题
- **验证**：先制定重构计划再执行
- **技能调用**：`Skill("architecture-analysis-specialist")`

## 🛡️ 风险检查清单

### 修改前必须检查：
- [ ] 是否已阅读CLAUDE.md相关章节？
- [ ] 问题类型是否明确？
- [ ] 是否有明确的影响范围分析？
- [ ] 是否准备回退方案？
- [ ] 是否进行过小范围测试？

### 修改后必须验证：
- [ ] 编译错误数量是否减少？
- [ ] 是否引入新的错误？
- [ ] git status是否显示预期变更？
- [ ] 相关测试是否通过？

## 🚨 危险信号识别

### 立即停止的信号：
1. **错误数量突然增加**
2. **出现编码格式问题**（BOM标记、乱码）
3. **Lombok相关错误爆发**
4. **依赖注入错误大量出现**
5. **感觉不确定时** - 立即停止，重新分析

### 安全信号：
1. **错误数量稳定减少**
2. **修改范围可控**
3. **编译输出清晰**
4. **测试结果正常**
