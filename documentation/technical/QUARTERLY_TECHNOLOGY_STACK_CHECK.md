## 🔍 检查内容

### 1. 技术栈版本一致性检查

**检查项**:
- ✅ 所有文档中的技术栈版本是否与标准规范一致
- ✅ 是否存在硬编码的技术栈版本
- ✅ 历史版本说明是否已正确标注

**检查脚本**:
```powershell
# 运行季度检查脚本
.\scripts\documentation\check-technology-stack-consistency.ps1 -GenerateReport
```

### 2. 技术栈标准引用检查

**检查项**:
- ✅ 新文档是否引用技术栈标准规范
- ✅ 包含技术栈描述的文档是否都引用了标准规范
- ✅ 引用格式是否正确

### 3. 文档更新检查

**检查项**:
- ✅ 技术栈标准规范是否已更新到最新版本
- ✅ 所有文档是否已同步更新
- ✅ 是否存在过时的技术栈描述

## 🔧 检查工具

### 自动化检查脚本

**脚本路径**: `scripts/documentation/check-technology-stack-consistency.ps1`

**使用方法**:
```powershell
# 运行检查（仅检查，不修改）
.\scripts\documentation\check-technology-stack-consistency.ps1 -GenerateReport

# 检查结果将生成报告文件
# documentation/technical/TECHNOLOGY_STACK_CONSISTENCY_CHECK_REPORT_YYYY-MM-DD_HHmmss.md
```

### 手动检查清单

1. **技术栈标准规范检查**
   - [ ] 确认 `TECHNOLOGY_STACK_STANDARD.md` 中的版本是否为最新
   - [ ] 对比 `microservices/pom.xml` 确认版本一致性

2. **文档引用检查**
   - [ ] 检查新文档是否引用技术栈标准
   - [ ] 检查包含技术栈描述的文档是否都引用了标准

3. **版本一致性检查**
   - [ ] 检查所有文档中的技术栈版本
   - [ ] 识别并修复版本不一致的文档

## 📝 检查流程

### 1. 准备阶段（检查前1周）

- [ ] 通知相关团队检查时间
- [ ] 准备检查工具和脚本
- [ ] 确认技术栈标准规范为最新版本

### 2. 执行阶段（检查周）

- [ ] 运行自动化检查脚本
- [ ] 生成检查报告
- [ ] 人工审查关键文档
- [ ] 识别需要修复的问题

### 3. 修复阶段（检查后2周）

- [ ] 修复P0级问题（必须修复）
- [ ] 修复P1级问题（建议修复）
- [ ] 更新相关文档

### 4. 验证阶段（修复后1周）

- [ ] 重新运行检查脚本验证修复结果
- [ ] 确认所有问题已修复
- [ ] 更新检查报告状态
