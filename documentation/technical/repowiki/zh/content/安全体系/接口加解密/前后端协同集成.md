## 概述

智能管理系统采用前后端分离架构，通过统一的加密框架实现数据传输的安全保护。该框架支持多种加密算法，包括国密SM4和AES，提供灵活的加密策略选择，确保敏感数据在传输过程中的安全性。

### 主要特性

- **多算法支持**：同时支持国密SM4和AES加密算法
- **透明化加密**：基于注解的自动加密解密机制
- **双向加密**：支持请求参数加密和响应数据加密
- **无缝集成**：与Spring Boot和Vue.js框架深度集成
- **高性能**：优化的加密算法实现，保证系统性能

## 加密算法支持

### AES加密算法

AES（Advanced Encryption Standard）是一种对称加密算法，具有以下特点：

| 特性 | 描述 |
|------|------|
| 密钥长度 | 支持128位、192位、256位 |
| 工作模式 | ECB（电子密码本模式） |
| 填充方式 | PKCS7填充 |
| 编码格式 | Base64编码 |
| 密钥要求 | 16字节（128位） |

### SM4加密算法

SM4是中国国家密码标准，专为中国国情设计：

| 特性 | 描述 |
|------|------|
| 密钥长度 | 128位固定 |
| 工作模式 | ECB模式 |
| 填充方式 | PKCS7填充 |
| 编码格式 | Base64编码 |
| 密钥要求 | 16字节（128位） |
| 兼容性 | 与Java UTF-8编码兼容 |

    @Cacheable(value = "encryptionCache", key = "#data")

        // 缓存加密结果，避免重复计算

## 故障排除

### 常见问题及解决方案

#### 1. 加密解密失败

**问题现象**：
- 前端报错："加密数据格式错误"
- 后端报错："解密失败"

**排查步骤**：
```mermaid
flowchart TD
A[加密解密失败] --> B{检查密钥一致性}
B --> |不一致| C[同步前后端密钥]
B --> |一致| D{检查编码格式}
D --> |不正确| E[调整编码方式]
D --> |正确| F{检查数据格式}
F --> |格式错误| G[修正数据结构]
F --> |格式正确| H[检查网络传输]
```

**解决方案**：
1. 确认前后端使用相同的加密算法
2. 检查密钥是否正确配置
3. 验证数据编码格式一致性

#### 2. 性能问题

**问题现象**：
- 接口响应时间过长
- 系统CPU使用率过高

**优化方案**：
1. 启用加密结果缓存
2. 使用异步加密处理
3. 优化加密算法参数

#### 3. 兼容性问题

**问题现象**：
- 不同浏览器加密结果不一致
- 移动端解密失败

**解决方法**：
1. 统一编码格式（推荐UTF-8）
2. 检查浏览器兼容性
3. 验证移动端网络环境
