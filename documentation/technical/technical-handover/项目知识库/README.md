## 📖 使用指南

### 1. 新团队成员入门路径

1. **了解项目**: 阅读 [项目介绍.md](01-项目概览/项目介绍.md)
2. **理解架构**: 学习 [整体架构设计.md](02-架构设计/整体架构设计.md)
3. **搭建环境**: 按照 [开发环境搭建.md](05-开发指南/开发环境搭建.md) 配置环境
4. **学习规范**: 掌握 [代码规范.md](05-开发指南/代码规范.md) 和 [API设计规范.md](05-开发指南/API设计规范.md)
5. **实践操作**: 完成 [实验手册.md](11-培训资料/实验手册.md) 中的练习

### 2. 开发人员常用路径

1. **API开发**: 参考 [API设计规范.md](05-开发指南/API设计规范.md)
2. **数据库设计**: 遵循 [数据库设计规范.md](05-开发指南/数据库设计规范.md)
3. **服务开发**: 查看对应服务文档（[核心服务](03-核心服务/)、[支撑服务](04-支撑服务/)）
4. **测试编写**: 学习 [测试指南.md](05-开发指南/测试指南.md)
5. **版本管理**: 使用 [Git工作流.md](05-开发指南/Git工作流.md)

### 3. 运维人员常用路径

1. **系统部署**: 遵循 [部署指南.md](06-运维指南/部署指南.md)
2. **监控配置**: 配置 [监控告警.md](06-运维指南/监控告警.md)
3. **日志分析**: 使用 [日志管理.md](06-运维指南/日志管理.md)
4. **备份恢复**: 执行 [备份恢复.md](06-运维指南/备份恢复.md)
5. **故障处理**: 参考 [故障处理.md](06-运维指南/故障处理.md)

## 📝 知识库维护

### 1. 文档更新原则

- **及时性**: 发现问题或改进建议时及时更新
- **准确性**: 确保文档内容准确无误
- **实用性**: 文档内容要有实际指导意义
- **完整性**: 文档结构完整，内容详实

### 2. 文档贡献流程

1. **发现问题**: 发现文档缺失或错误
2. **创建Issue**: 在项目管理系统中创建文档改进Issue
3. **编写文档**: 按照模板编写或修改文档
4. **审核验证**: 团队审核文档内容
5. **提交更新**: 合并到主分支并发布

### 3. 文档质量标准

- **格式规范**: 使用Markdown格式，遵循统一模板
- **内容准确**: 技术内容准确，经过验证
- **结构清晰**: 逻辑清晰，层次分明
- **实用性强**: 能够指导实际工作
